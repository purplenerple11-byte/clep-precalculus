Inverse Trig & the Principal Range

Why sin⁻¹(sin 2π/3) is not 2π/3 — and the one rule that makes every inverse-trig question mechanical.

Here's the trap this whole lesson exists to disarm: sin⁻¹(sin 2π/3) looks like the two operations should cancel and return 2π/3. They don't. The answer is π/3. If that surprises you, good — by the end of this page it won't, and this exact question shape is a CLEP favorite because it punishes anyone running on autopilot.

The one idea

sin⁻¹ does not mean "undo the sine." It means:

What arcsin actually promises "Give me the one angle in [−π/2, π/2] whose sine is this value." Sine hits every value infinitely many times — a function can only return one answer, so each inverse function commits to one agreed-upon window, called the principal range. Every inverse-trig question is really asking: which angle inside the window?
FunctionPrincipal rangeWhich quadrantsNegative inputs land…
sin⁻¹ x[−π/2, π/2]QI and QIVas a negative angle (QIV)
cos⁻¹ x[0, π]QI and QIIin QII, between π/2 and π
tan⁻¹ x(−π/2, π/2)QI and QIVas a negative angle (QIV)

Notice the pattern: all three include QI, so positive inputs are easy — they're the special-angle values you already know. The whole game is negative inputs. arcsin and arctan answer with a negative angle; arccos answers in the second quadrant. That asymmetry is where the traps live.

The trap itself: sin⁻¹(sin θ)

Now the composition. The inner function runs first and destroys information — it turns the angle into a bare number. The outer inverse then rebuilds an angle from that number, but it can only answer inside its window.

The two-line rule for f⁻¹(f(θ)) If θ is already in the principal range → it survives unchanged.
If not → compute the inner value, then find the in-range angle sharing it.
The other direction is friendlier: sin(sin⁻¹ x) = x for every x in [−1, 1], no exceptions — arcsin hands sine a legal angle, and sine doesn't care which window it came from.
